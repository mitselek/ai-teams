[LEARNED] **Scope vs ownership are orthogonal fields, not one field.** Scope answers "who within a tenant should read this?" (agent-only, team-wide, cross-team). Ownership answers "which tenant(s) authoritatively maintain this?" (owned-by-X, co-owned). An entry can be `scope: team-wide, ownership: co-owned-fr-apex` meaning "both tenants' teams should read it, both tenants authoritatively maintain it." Monte's framing made this legible; my earlier shorthand collapsed them.
[LEARNED] **Discipline primitives behave differently from provenance primitives in a substrate migration.** Provenance primitives (source-files, source-commits, source-issues) migrate as data — junction tables, no behavioral change. Enforcement primitives (ttl, revisit-by, confidence, status='disputed') have scheduled-scan or auto-promotion semantics that live in the Librarian's prompt and run against the filesystem today — each degrades to prompt-enforced shape on a DB substrate unless xireactor's schema adds first-class support. At n=4 enforcement primitives the regression is a quadruple hit. The distinction matters when scoping a migration: "we can fork xireactor's schema to add 4 fields" is a very different question from "we can throw these fields in a JSONB column."
